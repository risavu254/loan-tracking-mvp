What to Include

When reporting a vulnerability, please include:

  • Steps to reproduce the issue

üéØ Vulnerability Categories

In Scope

  • Authorization flaws
  • Data exposure/leakage
  • Cross-site scripting (XSS)
  • SQL injection
  • Cross-site request forgery (CSRF)

  • Missing security headers (low impact)
  • Rate limiting issues
  • Reports from automated scanners without manual verification

üîê Security Measures

Current Protections

  • Authentication: Firebase Authentication with email/password
  • Session Management: 8-hour session timeout with inactivity logout
  • Transport Security: HTTPS enforcement
  • Content Security Policy: Configured to prevent XSS
  • Domain Authorization: Restricted to authorized domains
  • Input Validation: Client-side and server-side validation
  • Error Handling: Secure error messages without information disclosure

Security Headers

  • X-Frame-Options: SAMEORIGIN
  • X-XSS-Protection: 1; mode=block
  • X-Content-Type-Options: nosniff
  • Referrer-Policy: strict-origin-when-cross-origin
  • Strict-Transport-Security: max-age=31536000
  • Permissions-Policy: geolocation=(), microphone=(), camera=()

üìú Responsible Disclosure

Please Follow Responsible Disclosure

  • Do not share vulnerability details publicly until resolved
  • Do not use automated scanners without permission
  • Respect user privacy and data
